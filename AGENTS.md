# PCS Core Agent

## 작업 시작

1. 모든 문서를 매번 읽지 않는다.
2. 먼저 `docs/ai/AI_INDEX.md`에서 작업 유형을 고른다.
3. 해당 유형의 필수 문서와 조건에 맞는 문서만 읽는다.
4. 기능 코드를 만들기 전에 `docs/features/{feature}.md`가 존재해야 한다.
5. 새 규칙이나 구현을 만들기 전에 원본 문서와 기존 공통 구현을 확인한다.
6. 읽은 문서는 최종 응답에 짧게 적는다.

구조, 계층, MyBatis, DTO, PageController 기준은 `docs/ai/pcs-project-structure-reference.md`를 따른다. 이 파일에서 같은 내용을 다시 정의하지 않는다.

## 로컬 서버 제어

- 사용자의 명시 요청 없이 애플리케이션 서버를 시작, 종료, 재시작하지 않는다.
- 서버는 사용자가 IntelliJ에서 제어한다.
- 파일 검사, 빌드, 문법 검사는 가능하지만 실행 중인 서버 프로세스는 조작하지 않는다.
- 서버 상태 변경이 필요하면 먼저 사용자에게 확인한다.

## 완료 조건

- 관련 기능 명세가 존재한다.
- 변경이 원본 규칙 문서와 일치한다.
- 필요한 최소 검증을 통과한다.
- 하네스 FAIL이 있으면 다음 작업 전에 해결한다.
- Codex Stop이 실패하면 `harness/reports/agent-failures.md`의 최신 FAIL을 수정하고 다시 검증한다.
- 검사를 우회하지 않는다. 해결할 수 없는 실패는 원인과 필요한 사용자 조치를 보고한다.
