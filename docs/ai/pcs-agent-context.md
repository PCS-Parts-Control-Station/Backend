# PCS 에이전트 컨텍스트

## 프로젝트 정체성

- 프로젝트명: 파츠관제소 / Parts Control Station / PCS
- 성격: 업체별 중고 PC 부품의 입고, 검수, 재고, 출고, 이력 관리 시스템
- 핵심 단위: 품목 마스터와 관리번호를 가진 개별 부품
- 기술 방향: Spring Boot, MyBatis, MariaDB, 정적 HTML/CSS/JavaScript, REST API
- AI는 서비스 기능이 아니라 개발 보조로만 사용한다.
- 하네스는 사람과 에이전트가 만든 결과를 검증한다.

구조와 계층 책임은 `docs/ai/pcs-project-structure-reference.md`, 작업 문서 선택은 `docs/ai/AI_INDEX.md`를 따른다.

## 업무 범위

포함:

- 회사 작업공간과 사용자
- 거래처, 품목 분류, 품목, 개별 관리번호
- 입고, 출고, 취소, 현재 재고
- 검수, 정정, 재검수, 검수 템플릿
- 등급, 판매 상태, 상태 변경 이력
- 업무용 통계와 집계 SQL

초기 범위 제외:

- 사진·엑셀 업로드, 바코드 스캔
- 결제, 배송, 채팅
- 중고 시세 수집
- 다중 창고
- 수리 이력과 캘린더 중심 기능
- 서비스 기능으로서의 AI

범위가 바뀌면 이 문서를 먼저 수정하고 관련 feature 문서를 연결한다.
