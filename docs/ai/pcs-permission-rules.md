# PCS Permission Rules

이 문서는 PCS 권한 기준의 원본 문서다.
기능 문서에서는 권한을 다시 길게 정의하지 않고 이 문서를 참조한다.

## 역할

```text
OWNER
ADMIN
STAFF
```

## OWNER

업체 작업공간의 최상위 소유자 계정이다.

- 회사 정보 조회/수정/활성 여부 변경
- 관리자 계정 생성과 복구
- 사용자 관리
- 거래처, 품목 분류, 품목 기준 관리
- 입고, 검수, 출고, 이력, 대시보드 접근
- 내 정보 관리

OWNER는 복구와 최종 관리 성격이 강하며, 일상 업무 계정으로 쓰는 것을 기본 흐름으로 보지 않는다.

## ADMIN

업체 업무를 관리하는 운영 관리자 계정이다.

- 사용자 생성/수정/임시 비밀번호 발급
- 거래처 관리
- 품목 분류 관리
- 품목 기준 관리
- 부품/재고 조회
- 입고, 검수, 출고 처리
- 입출고 취소
- 검수 정정/재검수
- 이력/대시보드 조회
- 내 정보 관리

## STAFF

실제 입고, 검수, 출고 업무를 처리하는 작업자 계정이다.

- 대시보드 조회
- 부품/재고 조회
- 입고 등록
- 검수 등록
- 출고 등록
- 이력 조회
- 내 정보 관리

STAFF는 사용자 관리, 거래처 관리, 품목 분류 관리, 품목 기준 관리, 회사 정보 변경을 할 수 없다.

## 적용 기준

- Controller에서 문자열로 권한을 직접 판단하지 않는다.
- 인증 사용자는 `PcsPrincipal`에서 받고, role 값은 프로젝트 Enum 기준으로 검증한다.
- 회사 범위 검증은 `docs/features/auth.md`의 `companyCode`, `companyId` 기준을 따른다.
- 기능 문서가 권한 기준을 적어야 하면 이 문서를 참조하고, 해당 기능에서 특별한 제한만 추가로 적는다.
