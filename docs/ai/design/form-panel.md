# Form Panel Design

등록/수정 입력 패널 기준이다.

## 적용 대상

- 품목 분류 등록/수정
- 사용자 생성/수정
- 거래처 생성/수정
- 품목 기준 등록
- 간단한 설정 변경

## 적용 제외

- 회사 등록처럼 화면 전체가 입력 폼인 공개 페이지
- 복잡한 상세 편집 화면
- 검수처럼 단계별 입력이 필요한 업무 화면
- 입고/검수/출고/이력의 업무 흐름 안내 패널

업무 흐름 안내 패널은 `docs/ai/design/workflow-panel.md`를 따른다.

버튼을 눌렀을 때 일시적으로 열리는 보조 입력이나 저장 확인은 `docs/ai/design/modal-dialog.md`를 따른다. 품목 분류 화면에서는 분류명/설명 입력은 오른쪽 드로어에 두고, 길어질 수 있는 사양 항목 입력만 모달로 분리한다.

## 기본 구조

오른쪽 등록/수정 폼 패널은 아래 구조를 기본으로 한다.

```text
side-panel
- panel-card
  - 제목
  - 짧은 설명
  - form
    - label + input
    - label + textarea
    - switch-row
    - form-actions
- panel-card.muted-panel
  - 운영 메모
```

## 오른쪽 작업 패널

목록을 보면서 바로 추가/수정해야 하는 관리 화면은 오른쪽 `side-panel`을 사용한다.

품목 관리와 품목 분류는 목록 전체 너비를 우선하므로 고정 `side-panel` 대신 헤더의 등록 버튼과 `management-detail-drawer`를 사용한다. 등록·상세·수정 모드와 입력 항목은 동일하게 유지한다.

기준:

- 기본 폭은 336px 전후로 둔다.
- 본문 목록보다 더 강하게 튀면 안 된다.
- 핵심 입력만 담는다.
- 길고 복잡한 입력은 별도 상세/등록 페이지로 분리한다.
- 품목 분류처럼 기본 입력은 짧지만 반복되는 하위 입력이 길어지는 경우, 기본 입력은 오른쪽 드로어에 유지하고 하위 입력만 모달로 분리한다.
- 품목 등록/수정처럼 분류에 따라 상세 사양 입력 항목이 달라지는 경우, 기본 입력은 품목 작업 드로어에 유지하고 `상세입력` 버튼으로 모달을 열어 사양 항목을 입력한다.

목록 행을 선택해 같은 오른쪽 영역에서 작업하는 관리 화면은 패널을 아래 모드로 전환한다.

```text
등록 모드
- 새 항목 등록 폼

상세 모드
- 선택한 항목 요약
- 상세 정보
- 수정
- 새 항목 등록

수정 모드
- 선택한 항목 수정 폼
- 저장 / 취소
- 새 항목 등록
```

기준:

- 최초 진입과 `새 항목 등록` 클릭 시 등록 모드를 보여준다.
- 목록 행 선택 시 상세 모드로 전환한다.
- 상세 모드의 `수정`을 누르면 같은 오른쪽 패널 안에서 수정 모드로 전환한다.
- `취소`는 선택 항목 상세 모드로 돌아간다.
- 등록/상세/수정 모드는 같은 패널 안에서 바뀌어야 하며, 별도 팝업처럼 튀지 않게 한다.
- 품목 관리 드로어는 닫기 버튼과 `Escape`를 지원하고, 닫힌 뒤 드로어를 연 버튼 또는 목록 행으로 포커스를 돌려준다.
- 관리형 작업 드로어는 본문 외부 클릭으로 닫되, 다른 목록 행 클릭은 닫기 동작으로 처리하지 않고 같은 드로어의 상세 내용만 교체한다.

공통 패널 전환 기준:

```text
panel-card.side-work-panel
- panel-mode[data-{domain}-panel="create"]
- panel-mode[data-{domain}-panel="detail"]
- panel-mode[data-{domain}-panel="edit"]
```

- `panel-mode`는 같은 `panel-card` 안에서 전환한다.
- 숨겨진 모드는 `hidden`을 사용한다.
- 상세/수정 모드 상단에는 `panel-title-bar`를 사용한다.
- 상세/수정 모드에서 새 등록으로 돌아가는 보조 버튼은 `panel-link-button`을 사용한다.
- `{domain}`에는 기능 도메인명을 사용한다. 예: 거래처 관리 화면은 `data-partner-panel`을 사용할 수 있다.

업무 흐름 안내, 전표 상세, 처리 단계 같은 오른쪽 패널은 이 문서에서 다시 정의하지 않고 `docs/ai/design/workflow-panel.md`를 따른다.

## 패널 카드

```css
.panel-card {
    padding: 20px;
    border: 1px solid var(--line);
    border-radius: var(--radius);
    background: rgba(255, 255, 255, 0.92);
    box-shadow: var(--shadow-sm);
}
```

기준:

- 제목은 20px 수준으로 둔다.
- 설명은 `--muted`, 14px 수준으로 둔다.
- 카드 안에 또 카드를 넣지 않는다.

## 입력 구성

단순 등록 폼 필드 예:

```text
분류명
설명
저장 / 초기화
```

품목 생성/수정 폼 필드 예:

```text
분류
품목명
제조사
제조사 모델명
상세입력
저장 / 초기화
```

기준:

- 분류는 가장 먼저 선택하게 둔다.
- 상세입력은 예상 단가, 안전 재고, 분류별 사양 항목을 모달에서 입력한다.
- 상세입력 버튼은 현재 입력된 상세값을 짧게 요약한다.
- 품목코드는 화면에서 입력받지 않고 저장 후 서버가 자동 생성한다.

거래처 생성/수정 폼 필드 예:

```text
거래처명
거래처 유형
거래 역할
상세 정보 입력
  - 연락처 선택
  - 이메일 선택
  - 주소 선택
  - 메모 선택
  - 거래 가능 상태로 등록
저장 / 초기화
```

기준:

- 필드는 세로로 배치한다.
- label은 13px, 굵게 표시한다.
- input/select는 48px 높이를 사용한다.
- textarea는 최소 112px 정도로 둔다.
- placeholder는 `--subtle`을 사용한다.
- 선택 입력값은 label 옆에 작은 `선택` 표시를 붙일 수 있다.
- 선택 표시는 `--subtle`, 11px 수준으로 작게 두고 입력 라벨보다 강하게 보이지 않게 한다.
- 거래처 등록에서 `거래처명`, `거래처 유형`, `거래 역할`은 필수 입력으로 둔다.
- 거래처 등록의 선택 입력은 DB 컬럼 기준으로 `연락처(phone)`, `이메일(email)`, `주소(address)`, `메모(memo)`를 둔다.
- 거래처 등록의 선택 입력은 처음부터 모두 펼치지 않고 `상세 정보 입력` 접힘 영역 안에 둔다.
- `상세 정보 입력` 문구는 접힘 박스 안에 넣지 않고 일반 입력 라벨처럼 위에 둔다.
- 접힘 박스 요약에는 `연락처, 이메일, 주소, 메모`처럼 선택 입력의 종류만 짧게 보여주고, 오른쪽 `+`/`-` 표시로 열림 상태를 구분한다.
- 거래처의 `거래 가능 상태` 체크는 상세 정보 접힘 영역 안에 둔다. 기본 등록 흐름에서는 필수 3개 입력 후 바로 저장할 수 있게 하고, 상태를 조정하고 싶을 때만 상세 정보 영역을 펼치게 한다.

선택 입력 접힘 구조:

```text
optional-field-group
- optional-field-label
- details.optional-fields
  - summary
    - 연락처, 이메일, 주소, 메모
  - optional-fields-body
    - 선택 입력들
    - switch-row
```

기준:

- `상세 정보 입력`은 접힘 박스 안 문구가 아니라 일반 입력 라벨처럼 위에 둔다.
- 접힘 요약은 선택 입력 종류만 짧게 보여준다.
- 각 선택 입력 라벨 옆에는 작은 `선택` 표시를 붙일 수 있다.
- 등록 모드는 접힘 상태로 시작할 수 있다.
- 수정 모드는 기존 값 확인이 필요하므로 펼친 상태로 시작할 수 있다.

상세 모드 표시:

```text
side-detail-card
- 주요 값
- detail-badge-row

detail-list
- 연락처
- 이메일
- 주소
- 메모
- 수정일
```

기준:

- 상세 카드의 주요 값은 18px 전후의 굵은 제목으로 둔다.
- 배지는 여러 개가 줄바꿈될 수 있게 `flex-wrap`을 허용한다.
- 상세 정보는 `detail-list`로 표시하고, 값이 길면 `overflow-wrap: anywhere`를 적용한다.
- 상세 모드의 주요 행동은 `수정` 하나를 우선한다.

## Switch Row

간단한 boolean 설정은 checkbox 기반 switch row로 처리할 수 있다.

```text
[checkbox] 거래 가능 상태로 등록
```

기준:

- 텍스트는 명령형보다 상태 설명형으로 쓴다.
- `accent-color`는 `--cyan`을 사용한다.
- 너무 많은 switch를 한 패널에 넣지 않는다.
- 품목 분류 등록/수정에는 `active` switch를 두지 않는다.

## Form Actions

기준:

- primary: 저장, 생성, 적용
- secondary: 초기화, 취소
- 버튼은 한 줄에 배치하되 좁아지면 2열 또는 1열 grid로 전환한다.
- destructive action은 같은 위치에 섞지 않는다.

## 보조 메모 패널

오른쪽 패널 아래에는 짧은 운영 메모를 둘 수 있다.

단순 등록 폼 운영 메모 예:

```text
분류명은 회사 안에서 중복 불가
품목에 연결된 분류는 삭제하지 않고 이름과 설명을 정리
```

거래처 생성/수정 폼 운영 메모 예:

```text
거래처명은 회사 안에서 중복 불가
공급처는 입고 전표, 고객은 출고 전표에서 선택 가능
거래 불가 상태는 신규 입출고 전표에서 제외
삭제 대신 거래 상태 변경으로 이력 보존
```

기준:

- 배경은 `--surface-soft`를 사용한다.
- 목록은 3~4개 이하로 유지한다.
- 기능 설명보다 입력 판단에 필요한 운영 규칙만 적는다.

## 반응형 기준

오른쪽 패널의 폭별 위치와 `content-grid` 전환 기준은 `docs/ai/design/responsive-layout.md`를 따른다.

이 문서에서는 폼 내부만 정의한다.

- form action은 좁은 화면에서 2열 또는 1열 버튼으로 전환할 수 있다.
- panel-card padding은 유지하되 화면 좌우 여백을 줄인다.

## 금지

- 오른쪽 패널에 긴 도움말을 많이 넣지 않는다.
- 모든 필드를 한 줄에 억지로 배치하지 않는다.
- 목록과 무관한 설정을 같은 패널에 섞지 않는다.
- 복잡한 검수 입력을 단순 side-panel 안에 욱여넣지 않는다.
