# Modal Dialog Design

등록/수정, 확인, 보조 입력을 화면 이동 없이 처리하는 모달 기준이다.

## 적용 대상

- 저장, 취소, 삭제, 중지처럼 실행 전 확인이 필요한 작업
- 입고 저장 전 요약 확인
- 목록 화면에서 간단한 등록/수정 입력을 받을 때
- 현재 작성 중인 화면 상태를 유지해야 하는 보조 입력
- 상세 페이지로 이동하기에는 입력 범위가 작은 작업

## 적용 제외

- 검색 필터, 정렬, 탭 전환처럼 즉시 반복 사용하는 조작
- 입력 필드가 많고 검토가 필요한 전체 등록 페이지
- 검수처럼 단계가 많고 화면 전체 맥락이 필요한 업무
- 긴 설명이나 도움말을 읽히기 위한 안내창

목록을 보면서 항상 열어두는 입력은 `form-panel.md`를 우선 사용한다. 모달은 사용자가 버튼을 눌렀을 때 일시적으로 열리는 작업에 사용한다.

## 기본 구조

HTML은 기본적으로 `<dialog>`를 사용한다.

```text
dialog
- modal-card
  - modal-header
    - page-kicker
    - h2
    - 짧은 설명
    - close button
  - modal-body
    - form fields 또는 summary
  - modal-actions
    - 취소
    - 주요 액션
```

기준:

- 제목은 작업 결과를 명확히 말한다.
- 설명은 한 문장으로 제한한다.
- 닫기 버튼은 우상단에 둔다.
- 하단 액션은 `취소`를 왼쪽, 주요 액션을 오른쪽에 둔다.
- 저장 중에는 주요 액션과 닫기/취소 버튼을 비활성화한다.

## 확인 모달

저장, 취소, 삭제, 중지처럼 되돌리기 어렵거나 DB 상태가 바뀌는 작업에 사용한다.

입고 저장 확인 예:

```text
입고 전표를 저장하시겠습니까?
저장 후 전표와 부품별 입고 이력이 생성됩니다.

거래처
입고 사유
부품 라인
총 수량

[취소] [저장]
```

기준:

- 제목은 `저장하시겠습니까?`처럼 짧고 직접적으로 쓴다.
- 본문에는 사용자가 마지막으로 확인해야 하는 핵심 값만 둔다.
- 요약 항목은 3~5개 이하로 유지한다.
- 입고/출고처럼 수량이 중요한 작업은 총 수량을 반드시 표시한다.
- 삭제/취소/중지 같은 위험 작업은 버튼 문구를 `삭제`, `취소 처리`, `사용 중지`처럼 구체적으로 쓴다.
- 위험 작업의 primary 색상은 별도 destructive 스타일이 정의되기 전까지 기존 primary와 섞지 않고 문구로 명확히 구분한다.

## 등록/수정 모달

현재 화면을 유지하면서 작은 입력을 받을 때 사용한다.

기준:

- 필수 입력을 최소화한다.
- 상세 입력은 접힘 영역 또는 별도 상세 화면으로 분리한다.
- 저장 후 현재 화면의 목록, 선택값, 요약을 즉시 갱신한다.
- 저장 실패 시 모달 안에 짧은 오류 메시지를 표시한다.
- 등록 API가 아직 없으면 화면에 임시 데이터만 추가하지 않는다.

## 화면 이동과 상태 보존

모달은 작성 중인 본문 상태를 잃지 않기 위해 사용한다.

기준:

- 모달을 열고 닫아도 본문 폼 값은 유지한다.
- 저장 확인 모달의 `취소`는 API를 호출하지 않는다.
- 저장 확인 모달의 `저장`에서만 API를 호출한다.
- 저장 성공 후 이동이 필요하면 불필요한 지연 없이 즉시 이동한다.
- 이동 후 완료 피드백은 도착 화면의 토스트나 상단 알림으로 표시한다.

## 토스트와의 구분

모달은 사용자 결정을 받는 UI이고, 토스트는 처리 결과를 잠깐 알리는 UI다.

기준:

- 실행 전 확인은 모달을 사용한다.
- 실행 후 성공 알림은 토스트를 사용할 수 있다.
- 토스트에는 버튼이나 복잡한 요약을 넣지 않는다.
- 토스트는 2~3초 뒤 자동으로 사라지게 한다.

## 반응형 기준

데스크톱:

- 화면 중앙에 둔다.
- 일반 입력 모달은 680px 전후를 넘지 않는다.
- 확인 모달은 520px 전후로 더 좁게 둔다.

모바일:

- 폭은 `calc(100vw - 24px)` 또는 `calc(100vw - 32px)`로 둔다.
- 본문 입력은 1열로 전환한다.
- 하단 버튼은 세로 1열 또는 동일 폭 2열로 배치한다.
- 텍스트가 버튼 안에서 줄바꿈되어도 높이가 안정적으로 유지되게 한다.

## 접근성 기준

- `<dialog>`에 `aria-labelledby`를 지정한다.
- 제목 `h2`의 id와 `aria-labelledby`를 연결한다.
- 닫기 버튼에는 `aria-label="닫기"`를 둔다.
- 상태 메시지는 필요하면 `role="status"`와 `aria-live="polite"`를 사용한다.
- `Escape`와 배경 클릭으로 닫을 수 있는지는 작업 위험도에 따라 결정한다.
- 저장 중에는 배경 클릭 또는 닫기 버튼으로 중복 제출 상태를 만들지 않는다.

## 금지

- 카드 안에 또 모달처럼 보이는 카드를 중첩하지 않는다.
- 모달 안에 긴 도움말을 넣지 않는다.
- 단순 필터 변경마다 확인 모달을 띄우지 않는다.
- 저장 확인 없이 API를 먼저 호출하지 않는다.
- API 없이 화면에만 임시 등록 데이터를 추가하지 않는다.
