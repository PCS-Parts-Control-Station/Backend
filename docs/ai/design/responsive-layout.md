# Responsive Layout Design

업무 화면의 breakpoint와 폭별 전환 시점을 정의한다. 컴포넌트 내용과 동작은 각 소유 문서를 따른다.

## 우선순위

1. 본문과 보조 영역이 겹치지 않는다.
2. 입력·목록의 핵심 값과 행동을 보존한다.
3. 좁은 화면에서 억지 압축보다 1열·카드형·가로 스크롤로 전환한다.
4. 사이드바와 드로어는 공통 동작을 유지한다.

## Breakpoint

| 폭 | 전환 |
|---|---|
| 전체 | 사이드바 기본 닫힘 off-canvas, 등록 화면은 본문+336px 보조 패널 가능 |
| 1280px 이하 | 필터가 많은 검색 폼은 2줄 가능 |
| 1180px 이하 | 등록·처리 `content-grid`를 1열로 전환 |
| 840px 이하 | 헤더 액션, 기본 검색 폼, 단순 목록 모바일 전환 |
| 640px 이하 | 전표형 목록 카드형 전환 |
| 560px 이하 | 설명과 버튼 폭 추가 압축 |

breakpoint는 viewport 기준이며 페이지별 임의 값을 추가하지 않는다.

## 사이드바

사이드바 구조와 열림 동작은 `workspace-layout.md`가 소유한다. 모든 폭에서 본문 밖 off-canvas를 유지하고 별도의 데스크톱 고정 사이드바 모드를 만들지 않는다.

## 헤더

- 중간 폭에서는 메뉴, 제목, 액션을 한 줄로 유지한다.
- 840px 이하에서는 액션을 다음 줄로 내린다.
- 모바일에서는 액션을 동일 폭 2열 또는 1열로 배치한다.
- 긴 설명은 두 줄을 넘기지 않는다.

## 본문과 보조 패널

- 등록·처리 화면은 넓은 폭에서 `minmax(0, 1fr) 336px`를 기본으로 한다.
- 1180px 이하에서 보조 패널을 본문 아래로 내린다.
- `content-main`, `side-panel`, 카드에는 필요한 `min-width: 0`을 보장한다.
- 조회 중심 화면은 처음부터 목록 전체 폭을 사용하고 상세는 드로어로 연다.

## 검색 폼

- 필터 2~3개는 가능한 폭에서 한 줄을 유지한다.
- 필터 4개 이상은 1280px 전후에서 검색어를 첫 줄 전체 폭, 나머지를 둘째 줄에 둘 수 있다.
- 840px 이하에서는 기본 1열로 전환한다.
- 화면 밖으로 밀리면 페이지 예외보다 공통 `filter-form`을 먼저 수정한다.

필터의 내용과 modifier는 `data-table.md`가 소유한다.

## 목록

- 단순 관리 목록은 840px 이하에서 카드형으로 전환한다.
- 전표·이력형 목록은 중간 폭에서 `min-width`와 내부 가로 스크롤을 허용한다.
- 전표형은 640px 이하에서 카드형으로 전환한다.
- 모바일 정보 순서와 라벨 기준은 `data-table.md`를 따른다.
- 헤더를 숨기는 카드형 목록은 각 값의 의미를 잃지 않게 한다.

## 드로어와 모달

- 드로어의 폭, 외부 클릭, focus, 스크롤은 `side-drawer.md`를 따른다.
- 데스크톱 420~520px, 태블릿 60~70%, 모바일 거의 전체 폭을 기본 범위로 한다.
- 모달의 모바일 전환은 `modal-dialog.md`를 따른다.

## 검증 폭

```text
1600px: 본문 2열과 off-canvas
1440px: 본문 2열 안정성
1280px: 검색 폼 전환
1100px: 본문 1열과 보조 패널 하단
390px: 메뉴·검색·목록·드로어 가로 넘침 없음
```

상호작용:

- 사이드바 backdrop·Escape·aria 상태
- 카드형 목록 정보 우선순위
- 열린 드로어에서 다른 행 선택
- 모바일 focus와 버튼 높이

## 금지

- 페이지별 다른 breakpoint와 햄버거 구조
- 본문 위에 workflow 패널 겹치기
- 모바일 검색 폼을 다열로 강제
- 라벨 없이 데이터만 세로 나열
- 드로어 오버레이로 목록 조작 차단
