# Dashboard Design

운영 상태와 처리 우선순위를 한 화면에서 판단하는 조회 허브 기준이다. 실제 지표와 이동 계약은 `docs/features/dashboard.md`가 원본이다.

## 구조

```text
workspace-header
dashboard-summary
priority-section
statistics-section
recent-section
```

- workflow 패널이나 입력 폼을 두지 않는다.
- 넓은 화면에서는 요약+통계, 우선 처리+최근 처리를 각각 2열로 구성할 수 있다.
- 태블릿 이하에서는 섹션을 세로로 쌓는다.

## 요약

- 상단 카드는 4개 내외의 가장 중요한 현재 값만 보여준다.
- 숫자, 짧은 라벨, 필요한 상태 신호로 구성한다.
- 보조 지표를 모두 카드로 올리지 않는다.

## 우선 처리

- 사용자가 지금 확인해야 할 상위 항목만 보여준다.
- 유형, 품목·분류, 수량, 이동 행동이 한 행에서 읽혀야 한다.
- 담당자 배정보다 상태와 처리 필요성을 우선한다.
- 빈 결과는 큰 빈 카드 대신 짧은 완료 문구를 사용한다.

## 통계

- 숫자와 간단한 비율·막대를 우선하고 차트를 남발하지 않는다.
- 상태별 색은 의미를 구분하되 위험색이 화면을 지배하지 않게 한다.
- 애니메이션은 한 번만 작게 실행하며 reduced motion을 지원한다.

## 최근 처리

- 처리 유형, 대상 요약, 수량, 처리일을 표시한다.
- 행 전체를 관련 업무나 이력 이동에 사용할 수 있다.
- 상세 분석은 해당 목록 화면으로 보낸다.

## 반응형

- 데스크톱: 주요 섹션 2열 가능
- 태블릿: 섹션 1열, 요약 2열
- 모바일: 요약 1열, 목록 카드형, 헤더 액션 1열 또는 2열
