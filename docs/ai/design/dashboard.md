# Dashboard Design

업무 대시보드 화면 기준이다.

## 적용 대상

- `/w/{companyCode}/dashboard`

## 적용 제외

- 일반 목록 화면
- 상세 화면
- 이력 타임라인 화면

## 목적

대시보드는 통계와 처리 우선순위를 한 곳에서 보여준다.

PCS에서 통계는 별도 메뉴로 흩어지기보다 대시보드에 통합한다. 관리자는 대시보드에서 오늘 무엇을 처리해야 하는지 바로 판단할 수 있어야 한다.

## 기본 구조

```text
workspace-layout
- workspace-sidebar
- workspace-main
  - workspace-header
  - summary-grid
  - priority-section
  - statistics-section
  - alert-section
```

## 요약 카드

상단 요약 카드는 오늘의 운영 상태를 보여준다.

예:

```text
오늘 입고
검수 대기
판매 가능
출고 예정
출고 차단
재고 부족
```

기준:

- 카드는 8px radius와 얇은 border를 사용한다.
- 숫자는 `--navy`로 강하게 표시한다.
- 상태성 숫자는 badge 또는 작은 상태 라벨을 함께 둔다.
- 카드 수가 많으면 2줄 grid로 배치한다.

## 우선 처리 목록

작업큐라는 별도 메뉴를 만들기보다 대시보드 안에서 우선 처리 목록으로 보여준다.

우선 처리 예:

- 검수 대기 수량이 많은 부품
- 재고 부족 부품
- 출고 차단 부품
- 오래 처리되지 않은 입고 건
- 정정 또는 재검수가 필요한 부품

기준:

- 담당자 배정보다 상태와 처리 필요성이 먼저 보여야 한다.
- 모든 부품을 나열하지 않는다.
- 관리자가 지금 확인해야 할 상위 항목만 보여준다.

## 통계

통계는 운영 판단에 필요한 수준만 둔다.

예:

- 기간별 입고/출고 수량
- 카테고리별 재고
- 검수 결과 분포
- 판매 가능/보류/차단 비율
- 불량률

기준:

- 차트가 많아져 화면을 복잡하게 만들지 않는다.
- 숫자와 간단한 막대/비율 표시를 우선한다.
- 상세 분석이 필요하면 목록 화면으로 이동하게 한다.

## 알림 영역

주의가 필요한 상태를 작은 목록으로 표시한다.

예:

```text
안전 재고 이하
검수 대기 7일 초과
출고 차단 상태
상태 정정 발생
```

기준:

- 빨강은 실제 위험/차단에만 사용한다.
- 일반 안내는 회색 또는 blue-soft로 표현한다.
- 알림은 액션 가능한 링크를 포함할 수 있다.
