# Detail Page Design

상세 화면 기준이다.

## 적용 대상

- `/w/{companyCode}/parts/{partId}`
- 개별 부품 상세
- 사용자 상세
- 거래처 상세
- 품목 분류 상세가 별도 페이지로 확장되는 경우

## 적용 제외

- 목록/검색 중심 화면
- 대시보드
- 이력 타임라인 중심 화면

## 목적

상세 화면은 한 대상을 깊게 확인하고 수정하는 화면이다.

PCS에서 특히 중요한 상세 화면은 부품 상세다. 부품 마스터 정보와 실제 개별 부품 unit 목록, 검수/판매/재고 상태가 연결되어 보여야 한다.

## 기본 구조

```text
workspace-layout
- workspace-main
  - detail-header
  - detail-summary
  - detail-content
    - main-detail-area
    - side-summary-panel
```

## 상세 헤더

포함 정보:

- 대상 이름
- 보조 식별값
- 현재 상태
- 주요 액션

부품 상세 예:

```text
RTX 3060 Twin Edge
part_code: GPU-3060-ZOTAC
활성 / 판매 가능 수량 / 검수 대기 수량
[수정] [비활성화]
```

기준:

- 제목은 `workspace-header h1`보다 약간 작거나 같은 수준으로 둔다.
- 식별값은 code chip으로 표현할 수 있다.
- destructive action은 primary 옆에 섞지 않는다.

## 요약 영역

상세 상단에는 빠른 판단을 위한 요약을 둔다.

부품 상세 요약 예:

- 현재 재고
- 검수 대기
- 판매 가능
- 출고 차단
- 안전 재고

기준:

- 숫자는 크게 표시한다.
- 상태는 badge로 표현한다.
- 요약 카드는 3~5개 이내로 유지한다.

## 상세 탭

상세 정보가 많아지면 탭으로 분리한다.

부품 상세 탭 예:

```text
개별 부품
입출고 이력
검수 이력
상태 변경 이력
설정
```

기준:

- 탭은 실제 화면 전환 단위일 때만 사용한다.
- 탭 안에 또 탭을 넣지 않는다.
- 이력성 탭은 `history-timeline.md` 기준을 따른다.

## 개별 부품 목록

부품 상세의 핵심은 개별 부품 unit이다.

표시 정보:

- 관리번호
- 제조사 시리얼
- 검수 상태
- 등급
- 판매 상태
- 현재 unit 상태
- 최근 이력

기준:

- 관리번호는 첫 번째 열 또는 카드 상단에 둔다.
- 상태값은 모두 badge로 표현한다.
- 동일 모델이라도 unit별 상태가 다르다는 점이 드러나야 한다.

## 오른쪽 요약 패널

오른쪽 패널은 전체 맥락을 보조한다.

예:

- 기본 정보
- 품목 분류
- 제조사
- 안전 재고
- 등록자/수정일

기준:

- 편집 폼을 너무 많이 넣지 않는다.
- 상세 편집은 별도 모드 또는 모달/페이지로 분리할 수 있다.
- 패널은 정보 스캔용으로 유지한다.

## 반응형 기준

- 좁은 화면에서는 오른쪽 요약 패널을 본문 아래로 내린다.
- 요약 카드는 1~2열로 줄인다.
- unit 목록은 모바일 카드형 행으로 전환한다.
