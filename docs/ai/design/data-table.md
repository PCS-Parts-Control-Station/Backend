# Data Table Design

검색/필터/목록 테이블 화면 기준이다.

## 적용 대상

- `/w/{companyCode}/categories`
- `/w/{companyCode}/parts`
- `/w/{companyCode}/users`
- `/w/{companyCode}/partners`
- `/w/{companyCode}/inbound`
- `/w/{companyCode}/outbound`
- `/w/{companyCode}/history`의 목록형 영역

## 적용 제외

- 대시보드 요약 카드
- 상세 화면의 내부 정보 카드
- 타임라인 중심 이력 화면

## 기본 구조

검색과 목록이 함께 있는 업무 화면은 아래 구조를 기본으로 한다.

```text
content-main
- filter-card
- table-card
```

오른쪽 등록/수정 패널이 함께 있으면 `form-panel.md`도 같이 적용한다.

공통 modifier 기준:

```text
단순 관리 목록: simple-management-data-row
선택형 관리 목록: management-filter-form, inline-summary-header, management-data-row
전표형 목록: document-filter-form, inline-summary-header, document-data-table, document-data-row
```

modifier class는 새 규칙을 뜻하지 않는다. 기본 구조를 유지한 상태에서 화면 유형별 grid, 요약 배치, 행 폭만 조정하는 데 사용한다.

CSS 기준:

- 선택형 관리 목록은 `components/components.css`의 공통 목록 스타일을 우선 사용한다.
- 전표형 목록의 공통 구조는 `components/components.css`, 업무 흐름 표현은 `components/workflow.css`, 화면 고유 보강은 `pages/{page}.css`에 둔다.

## 검색 / 필터 카드

검색 카드는 목록 위에 둔다.

구조:

```text
filter-card
- 제목
- 짧은 설명
- filter-form
  - 검색어
  - 상태 필터
  - 검색 버튼
```

기준:

- 제목은 현재 목록을 좁히는 목적을 말한다.
- 설명은 한 줄 수준으로 짧게 둔다.
- 검색 버튼은 primary를 사용한다.
- 필터가 2~3개일 때는 한 줄 grid가 가능하다.
- 필터가 4개 이상이면 검색어 입력칸을 우선 넓게 두고, 선택 필터와 검색 버튼은 고정 범위 안에서 줄어들게 한다.
- 폭별 줄바꿈 시점은 `docs/ai/design/responsive-layout.md`를 따른다.

필터가 적은 화면 예:

```text
분류 검색
분류명과 설명으로 목록을 좁혀 봅니다.
검색어 / 검색
```

필터가 많은 화면 예:

```text
거래처 검색
거래처명, 유형, 역할, 거래 상태로 목록을 좁혀 봅니다.
검색어 / 유형 / 역할 / 거래 상태 / 검색
```

전표형 목록 검색 예:

```text
입고 전표 검색
전표번호, 거래처, 품목명, 처리 상태로 입고 내역을 좁혀 봅니다.
검색어 / 거래처 / 상태 / 검색
```

## 목록 테이블 카드

목록은 `table-card` 안에 둔다.

구조:

```text
table-card
- table-header
  - 제목
  - 설명
  - list-summary-box
- data-table
  - table-head
  - data-row
```

기준:

- 테이블 상단에는 제목과 설명을 둔다.
- 오른쪽에는 목록 현황 요약 박스를 둘 수 있다.
- 요약 박스는 `전체`, `거래 가능`, `거래 불가`, `템플릿`처럼 목록 판단에 필요한 숫자만 둔다.
- 데이터 행은 얇은 구분선으로 나눈다.
- 열 개수가 많으면 불필요한 열을 줄이고 상세 페이지로 넘긴다.

## 목록 요약

목록 요약은 숫자의 업무 중요도에 따라 박스형 또는 인라인형을 사용한다. 페이지 이름이 아니라 요약 정보가 사용자의 판단에 얼마나 직접적인지로 선택한다.

### 박스형 요약

요약 숫자가 목록 판단의 핵심이면 박스형 요약을 쓴다.

```text
목록 현황
전체 / 거래 가능 / 거래 불가 / 템플릿
짧은 운영 설명
```

기준:

- 숫자는 `--navy`로 강조한다.
- 라벨은 `--muted`로 작게 표시한다.
- 설명은 12px 수준으로 짧게 둔다.
- 요약 박스는 테이블 헤더 오른쪽에 두되, 좁아지면 아래로 내려간다.

### 인라인 요약

요약 숫자가 보조 정보이면 박스 배경 없이 인라인으로 둔다.

```text
전체 12 | 공급처 5 | 고객 4 | 거래 가능 10
```

기준:

- `목록 현황` 라벨과 운영 설명 문구는 넣지 않는다.
- 배경, border, 큰 숫자 강조를 제거한다.
- 라벨과 숫자는 본문 설명문과 비슷한 14px, `--muted`, 400 weight로 둔다.
- 항목 사이는 얇은 세로 구분선으로만 나눈다.
- 데스크톱과 태블릿 폭에서는 테이블 헤더 오른쪽 끝에 붙인다.
- 모바일 폭에서는 제목 아래로 내려가며, 매우 좁은 폭에서만 2열 요약으로 전환한다.

공통 modifier 기준:

```text
table-header.inline-summary-header
- list-summary-box
  - list-summary
  - list-summary.summary-compact
```

- `list-summary-box`는 기존 박스형 클래스명을 유지하더라도, 위 modifier 안에서는 border/background/padding을 제거한 인라인 요약으로 표시한다.
- 요약 항목은 4개 전후로 제한한다.
- 요약 항목이 3개 이하이면 `summary-compact`를 함께 사용해 항목 사이가 과하게 벌어지지 않게 한다.

## 데이터 행

행 구조:

```text
주요 값
설명
상태
등록자/처리자
수정일/처리일
관리 액션
```

기준:

- 첫 번째 열은 가장 중요한 값이다.
- 상태는 `.badge`로 표현한다.
- 행별 액션을 쓰는 경우 작은 outline 버튼을 사용한다.
- 행별 액션은 `수정`, `상세`처럼 짧은 명령어로 둔다.
- 상태 변경은 행마다 반복하기보다 오른쪽 패널이나 상세 화면에서 처리한다.
- 삭제보다 상태 변경으로 이력을 보존하는 흐름을 우선한다.
- 관리 목록의 데이터 행은 같은 목록 안에서 높이가 들쭉날쭉해지면 안 된다.
- 긴 이름, 코드, 모델명처럼 길이가 예측되지 않는 값은 목록 행 안에서 한 줄 말줄임으로 처리한다.
- 목록에서 말줄임된 전체 값은 오른쪽 상세 패널, 상세 화면, title 속성 같은 보조 경로로 확인할 수 있게 한다.
- 긴 텍스트를 표시하기 위해 JS에서 원본 값을 잘라 저장하지 않는다. 화면 표시 안정성은 CSS의 `white-space: nowrap`, `overflow: hidden`, `text-overflow: ellipsis`로 처리한다.

단순 관리 목록 행 예:

```text
분류명
설명
품목 수
수정일
```

공통 modifier 기준:

```text
data-row.simple-management-data-row
```

- 단순 관리 목록은 4열 전후의 적은 정보만 보여준다.
- 행 안에 상세 액션이 없거나 오른쪽 패널에서 처리하는 경우 사용한다.
- 품목 분류 목록은 상태 열을 표시하지 않는다.
- 품목 분류 목록은 해당 분류에 연결된 품목 수를 보조 정보로 보여준다.
- 모바일 전환 시 `수정일` 라벨이 `등록자`로 잘못 보이지 않게 별도 라벨 기준을 둔다.

품목 관리 목록 행 예:

```text
품목명
제조사 / 모델
분류
현재 재고
안전 재고
```

기준:

- 품목 목록은 품목명 아래에 자동 생성된 품목코드를 작게 표시할 수 있다.
- 제조사와 모델명은 한 열 안에서 함께 보여준다.
- 현재 재고가 안전 재고보다 낮으면 `재고 부족`으로 표시한다.
- 품목 목록에서는 active 상태를 반복 표시하지 않는다.
- 품목명, 품목코드, 제조사, 제조사 모델명, 분류명은 행 높이를 늘리지 않도록 한 줄로 고정하고 넘치는 값은 말줄임 처리한다.
- 품목 관리 목록 행은 데스크톱 목록에서 고정 높이를 유지한다. 모바일 카드형 전환 구간에서는 카드 구조에 맞춰 높이 고정을 해제할 수 있다.

공통 modifier 기준:

```text
data-row.part-management-data-row
```

거래처 관리 목록 행 예:

```text
거래처명
유형
역할
거래 상태
연락처
수정일
```

기준:

- 첫 번째 열의 주요 값은 굵게 표시한다.
- 유형, 역할, 거래 상태처럼 빠른 구분이 필요한 값은 배지로 표시한다.
- 연락처 같은 보조 값은 본문보다 강하게 보이지 않게 둔다.
- 행을 선택해 오른쪽 패널에서 상세/수정/상태 변경을 처리하는 화면은 줄별 `수정/제한/재개` 버튼을 반복하지 않는다.
- 선택형 행은 클릭 가능한 행으로 만들고, 선택된 항목의 상세/수정/상태 변경은 오른쪽 작업 패널에서 처리한다.
- 선택된 행은 왼쪽 accent line과 옅은 배경으로 표시한다.

공통 modifier 기준:

```text
data-row.management-data-row
data-row.management-data-row.is-selectable
data-row.management-data-row.is-selected
```

- 선택 가능한 행은 hover에서 옅은 blue 배경을 사용한다.
- 선택된 행은 왼쪽 3px accent line과 옅은 blue 배경을 사용한다.
- 선택형 관리 목록은 행 내부에 수정/상태 변경 버튼을 반복하지 않는다.

전표형 목록 행 예:

```text
전표번호
입고 내용
수량
상태
입고일
관리
```

공통 modifier 기준:

```text
data-table.document-data-table
data-row.document-data-row
data-row.document-data-row.is-selected
data-row.document-data-row.is-created
```

- 전표번호는 monospace 계열로 작고 강하게 표시한다.
- 전표형 행은 중간 폭에서 `min-width`를 유지하고 테이블 내부 가로 스크롤을 허용한다.
- 선택된 전표 행은 옅은 blue 배경으로 표시한다.
- 방금 생성된 전표 행은 더 연한 primary 배경으로 구분할 수 있다.
- 관리 버튼은 행 클릭 상세 열기와 충돌하지 않게 이벤트를 분리한다.

## 상태 배지

목록 화면에서 상태값은 텍스트만 쓰지 않는다.

active 상태 배지:

```text
사용 중: green
사용 안 함: gray
```

거래처 타입/역할/상태 배지 예:

```text
기업: blue
PC방: orange
개인: gray
공급처: blue
고객: blue
공급+고객: green
거래 가능: teal
거래 불가: gray
```

부품/재고 상태 배지 예:

```text
검수완료: green
검수대기: orange
판매가능: blue
판매보류: orange
불량: red
출고차단: red
```

## 페이징

- 서버 페이징을 쓰는 목록은 테이블 하단에 페이지 정보를 둔다.
- 페이지 정보는 `현재 페이지 / 전체 페이지 · 총 N건`처럼 짧게 표시한다.
- 페이지 이동은 `이전`, `다음` 버튼을 기본으로 두고, 불가능한 방향은 disabled 상태로 둔다.
- 페이징 동작과 응답 구조는 `docs/ai/pcs-pagination-rules.md`를 따른다.
- 행 클릭으로 오른쪽 상세 패널을 여는 화면은 선택된 행을 시각적으로 표시한다.
- 행 안의 `상세`, `취소` 같은 버튼 클릭은 행 클릭 상세 열기와 충돌하지 않게 이벤트를 분기한다.

## 긴 식별자 표시

전표번호, 관리번호처럼 긴 식별자는 원본 값을 유지하되 목록에서 시각적 무게를 낮춘다.

기준:

- API와 검색은 전체 식별자를 사용한다.
- 목록 셀에서는 작은 `ui-monospace` 계열 글꼴을 사용할 수 있다.
- `overflow-wrap: anywhere`로 좁은 폭에서 레이아웃을 밀지 않게 한다.
- 임의 축약은 상세/복사 흐름이 준비된 뒤에만 적용한다.

전표형 상세 패널이나 라인 목록에서 관리번호가 반복 노출되는 경우:

- 긴 값은 `code` 또는 monospace 셀로 보여준다.
- 복사 버튼은 작은 outline 버튼으로 둔다.
- 목록이 길어지면 내부 스크롤 또는 접기/펼치기를 제공한다.

## 모바일 전환

업무 화면 전체 반응형 기준과 breakpoint는 `docs/ai/design/responsive-layout.md`를 따른다. 이 문서는 테이블 행이 모바일에서 어떤 정보를 우선 보여줄지만 정의한다.

단순 관리 목록:

- 테이블 헤더는 숨긴다.
- `data-row`는 카드형 세로 행으로 전환한다.
- 각 cell 앞에는 작은 라벨을 붙일 수 있다.
- 행별 액션은 행 하단에 배치한다.

필터가 많은 화면은 태블릿 폭에서 검색 폼을 바로 1열로 내리지 않고 2줄 배치를 유지할 수 있다.

```text
검색어
유형 / 역할 / 거래 상태 / 검색
```

인라인 목록 요약은 태블릿 폭에서는 오른쪽 정렬을 유지하고, 모바일 폭에서만 제목 아래로 내린다.

단순 관리 목록은 `data-label` 또는 `::before`로 모바일 라벨을 붙일 수 있다.

전표형 목록은 중간 폭에서 카드형으로 바로 바꾸지 않고 `overflow-x: auto`와 행 `min-width`를 사용할 수 있다. 매우 좁은 폭에서는 카드형 행으로 전환해 가로 스크롤 의존을 줄인다.

전표형 모바일 행은 전표 1건이 한눈에 들어오도록 아래 순서를 우선한다.

```text
전표번호 / 상태
입고 내용
수량 / 입고일
관리 버튼
```

이 경우 `전표번호`, `입고 내용`, `관리` 같은 라벨은 반복 노출하지 않고, `수량`, `입고일`처럼 값만 보면 의미가 약한 항목만 작은 라벨을 붙인다.

## 빈 목록 / 로딩 / 오류 상태

목록 화면은 아래 상태를 고려한다.

- 검색 결과 없음
- 첫 데이터 없음
- 로딩 중
- API 오류
- 직전 생성/처리 결과 안내

기준:

- 빈 목록, 로딩, 오류 상태는 데이터 행과 구분되는 안내 행으로 표시한다.
- 상태 행이 단독으로 보이는 경우 안내 문구는 가운데 정렬하고 15px 전후의 굵은 글씨로 표시할 수 있다.
- 상태 행은 기본 HTML에서 `hidden`으로 시작하고 JS가 필요한 상태에서만 표시한다.
- 빈 목록은 테이블 내부 중앙 또는 카드 하단에 표시한다.
- 문구는 짧게 쓰고, 필요한 경우 primary action을 제공한다.
- 오류는 빨강을 과하게 쓰지 말고 재시도 버튼을 둔다.

직전 생성/처리 결과 안내:

```text
created-document-notice
```

- 목록 카드 위, 검색 카드 아래에 둔다.
- 기본 상태는 `hidden`으로 두고 필요한 경우 JS가 표시한다.
