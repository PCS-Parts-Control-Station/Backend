# Data Table Design

검색/필터/목록 테이블 화면 기준이다.

## 적용 대상

- `/w/{companyCode}/categories`
- `/w/{companyCode}/parts`
- `/w/{companyCode}/users`
- `/w/{companyCode}/partners`
- `/w/{companyCode}/history`의 목록형 영역

## 적용 제외

- 대시보드 요약 카드
- 상세 화면의 내부 정보 카드
- 타임라인 중심 이력 화면

## 기본 구조

카테고리 관리 화면에서 확정된 구조를 기준으로 한다.

```text
content-main
- filter-card
- table-card
```

오른쪽 등록/수정 패널이 함께 있으면 `form-panel.md`도 같이 적용한다.

## 검색 / 필터 카드

검색 카드는 목록 위에 둔다.

구조:

```text
filter-card
- 제목
- 짧은 설명
- filter-form
  - 검색어
  - 상태 필터
  - 검색 버튼
```

기준:

- 제목은 현재 목록을 좁히는 목적을 말한다.
- 설명은 한 줄 수준으로 짧게 둔다.
- 검색 버튼은 primary를 사용한다.
- 필터가 2~3개일 때는 한 줄 grid가 가능하다.
- 필터가 많아지면 2줄 또는 접힘 영역을 고려한다.

카테고리 화면 예:

```text
카테고리 검색
카테고리명, 설명, 사용 여부로 목록을 좁혀 봅니다.
검색어 / 사용 여부 / 검색
```

## 목록 테이블 카드

목록은 `table-card` 안에 둔다.

구조:

```text
table-card
- table-header
  - 제목
  - 설명
  - list-summary-box
- data-table
  - table-head
  - data-row
```

기준:

- 테이블 상단에는 제목과 설명을 둔다.
- 오른쪽에는 목록 현황 요약 박스를 둘 수 있다.
- 요약 박스는 `전체`, `사용`, `중지`, `템플릿`처럼 목록 판단에 필요한 숫자만 둔다.
- 데이터 행은 얇은 구분선으로 나눈다.
- 열 개수가 많으면 불필요한 열을 줄이고 상세 페이지로 넘긴다.

## 목록 요약 박스

카테고리 화면 기준:

```text
목록 현황
전체 / 사용 / 중지 / 템플릿
짧은 운영 설명
```

기준:

- 숫자는 `--navy`로 강조한다.
- 라벨은 `--muted`로 작게 표시한다.
- 설명은 12px 수준으로 짧게 둔다.
- 요약 박스는 테이블 헤더 오른쪽에 두되, 좁아지면 아래로 내려간다.

## 데이터 행

행 구조:

```text
주요 값
설명
상태
등록자/처리자
수정일/처리일
관리 액션
```

기준:

- 첫 번째 열은 가장 중요한 값이다.
- 상태는 `.badge`로 표현한다.
- 행별 액션은 작은 outline 버튼을 사용한다.
- 행별 액션은 `수정`, `중지`, `사용`, `상세`처럼 짧은 명령어로 둔다.
- 삭제보다 비활성화/중지 표현을 우선한다.

## 상태 배지

목록 화면에서 상태값은 텍스트만 쓰지 않는다.

카테고리 화면 기준:

```text
사용 중: green
사용 안 함: gray
```

부품/재고 화면은 공통 배지 기준을 확장한다.

```text
검수완료: green
검수대기: orange
판매가능: blue
판매보류: orange
불량: red
출고차단: red
```

## 모바일 전환

업무 화면 전체 반응형 기준은 `docs/ai/design/responsive-layout.md`를 따른다.

`max-width: 840px` 이하:

- `filter-form`은 1열로 전환한다.
- 테이블 헤더는 숨긴다.
- `data-row`는 카드형 세로 행으로 전환한다.
- 각 cell 앞에는 작은 라벨을 붙일 수 있다.
- 행별 액션은 행 하단에 배치한다.

카테고리 화면처럼 `::before`로 모바일 라벨을 붙일 수 있다.

입고 전표처럼 열 구분이 중요한 목록은 중간 폭에서 카드형으로 바로 바꾸지 않고 `overflow-x: auto`와 행 `min-width`를 사용할 수 있다. 단, `640px` 이하에서는 카드형 행으로 전환해 가로 스크롤 의존을 줄인다.

입고 전표 모바일 행은 전표 1건이 한눈에 들어오도록 아래 순서를 우선한다.

```text
전표번호 / 상태
입고 내용
수량 / 입고일
관리 버튼
```

이 경우 `전표번호`, `입고 내용`, `관리` 같은 라벨은 반복 노출하지 않고, `수량`, `입고일`처럼 값만 보면 의미가 약한 항목만 작은 라벨을 붙인다.

## 빈 목록 / 로딩 / 오류 상태

목록 화면은 아래 상태를 고려한다.

- 검색 결과 없음
- 첫 데이터 없음
- 로딩 중
- API 오류

기준:

- 빈 목록은 테이블 내부 중앙 또는 카드 하단에 표시한다.
- 문구는 짧게 쓰고, 필요한 경우 primary action을 제공한다.
- 오류는 빨강을 과하게 쓰지 말고 재시도 버튼을 둔다.
