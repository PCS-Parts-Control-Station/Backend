회사 등록 후 발급받은 업체 코드로 전용 업무 페이지에 들어갑니다.

관리번호 PCS-CPU-000128

현재 위치: 검수 진행

01 입고 등록

관리번호 발급 완료

02 검수 진행

B등급, 판매 보류

03 재고 대기

검수 확정 후 반영

04 출고 대기

전표 생성 전

업무 흐름

업무 화면은 부품의 상태 변화 순서로 나뉩니다.

  1. 01 입고

    실물 부품에 업체 내부 관리번호를 부여합니다.

  2. 02 검수

    카테고리 기준으로 등급과 판매 가능 상태를 결정합니다.

  3. 03 재고

    개별 부품 상태와 현재 수량을 함께 봅니다.

  4. 04 출고

    출고 전표와 실제 개별 부품 이동을 연결합니다.

  5. 05 이력

    누가, 언제, 왜 상태를 바꿨는지 남깁니다.

관리 포인트

관리자가 매일 확인하는 정보만 전면에 둡니다.

unit.serial 관리번호 기준 추적

개별 부품의 현재 상태와 처리 과정을 한 기준으로 확인합니다.

inspection.result 검수 결과 반영

검수 결과가 등급, 판매 가능 여부, 상태 이력으로 이어집니다.

stock.quantity 재고 정합성

부품별 현재 수량과 실제 개별 부품 수량이 맞는지 확인합니다.

status.history 상태 변경 이력

변경자, 변경 시점, 변경 사유를 이력으로 남깁니다.
